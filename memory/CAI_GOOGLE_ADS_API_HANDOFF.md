### 2. OAuth 2.0 Refresh Token

This is the one credential that has to be generated by walking through Google's
consent screen once. It gives our backend permission to call the Ads API on
behalf of the Google account that owns the MCC. Only needs to be done ONCE.

**Prerequisite:** you'll need the Client Secret from step 1 above.

**Step-by-step:**

1. Open Google Cloud Console → Credentials → click the OAuth Client
   (`1088577266472-…apps.googleusercontent.com`).
2. Under **"Authorized redirect URIs"**, click **+ Add URI** and paste:
   `https://developers.google.com/oauthplayground`
3. Click **Save**.
4. Go to https://developers.google.com/oauthplayground/
5. Click the **⚙️ gear icon** (top right) → check
   **"Use your own OAuth credentials"**.
6. Paste the Client ID and the Client Secret from step 1 → click **Close**.
7. In the left panel, in **"Input your own scopes"**, paste EXACTLY:
   `https://www.googleapis.com/auth/adwords`
8. Click **Authorize APIs**.
9. Sign in with a Google account that has access to the MCC `3947028100`.
   (Use the same Google login that manages the account normally.)
10. On the consent screen, click **Allow**.
11. Back in Playground, click **Exchange authorization code for tokens**.
12. Copy the **Refresh token** — it's a long string starting with `1//…`.
13. (Optional cleanup) After you have the refresh token, go back to Cloud
    Console → Credentials → OAuth Client → remove the
    `https://developers.google.com/oauthplayground` redirect URI. It was only
    needed for this one-time flow.
